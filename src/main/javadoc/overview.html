This document is the API specification for the FEST "Flexible Assertions" Module.

Currently, this module provides assertions that apply to the following data types:

  1. {@code Object}
  2. {@code String}
  3. {@code Collection}
  4. {@code List}
  5. {@code Map}
  6. Primitives ({@code boolean}, {@code int}, {@code char}, etc.)
  7. {@code Object} arrays
  8. Arrays of primitives
  9. {@code BufferedImage}
  10. {@code Throwable}
  11. {@code File}
  12. {@code BigDecimal}

Quick example: 

int removed = employees.removeFired();
{@link org.fest.assertions.Assertions#assertThat(int) assertThat}(removed).{@link org.fest.assertions.IntAssert#isZero isZero}();
 
List<Employee> newEmployees = employees.hired(TODAY);
{@link org.fest.assertions.Assertions#assertThat(java.util.Collection) assertThat}(newEmployees).{@link org.fest.assertions.CollectionAssert#hasSize(int) hasSize}(6);

Assertions can also be extended by using custom {@link org.fest.assertions.Condition}s. The following condition verifies that the characters in a {@code String} are in uppercase: 

class UpperCaseCondition extends {@link org.fest.assertions.Condition Condition}<String> {
  public boolean matches(String value) {
    if(isEmpty(value)) return false;
    return value.equals(value.toUpperCase());
  }

  public static UpperCaseCondition isUpperCase() {
    return new UpperCaseCondition("Uppercase");
  }
}
This example shows how to use such condition: 
{@link org.fest.assertions.Assertions#assertThat(java.lang.String) assertThat}("hello").as("Greeting").{@link org.fest.assertions.StringAssert#satisfies(org.fest.assertions.Condition) satisfies}(isUppercase());
which will fail with the message "{@code [Greeting] expected:Uppercase but was:<'hello'>}"
